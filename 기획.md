# PRD: Project "StoryWeaver" (가칭)

**AI 기반 웹소설 창작 코파일럿(Co-pilot) 플랫폼**

## 1. 개요 (Executive Summary)

- **제품 정의:** 작가가 설정한 세계관과 캐릭터를 AI가 기억하고, 플롯 구상부터 문장 집필까지 전 과정을 보조하는 웹 기반 저작 도구.
- **해결하려는 문제:**
    - 장편 연재 시 발생하는 설정 오류 및 개연성 붕괴.
    - 작가의 창작 고갈(Writer's Block) 및 반복 작업으로 인한 피로.
    - 단순 텍스트 생성 툴(ChatGPT 등)의 낮은 문맥 유지력(Long-context) 한계.
- **핵심 가치:** "기억하는 AI", "장르 문법을 아는 조수", "구조적인 글쓰기".

## 2. 타겟 유저 (Target Audience)

1. **메인 타겟:** 웹소설 작가 지망생 및 아마추어 작가 (아이디어는 있으나 필력/끈기가 부족한 층).
2. **서브 타겟:** 다작(多作)을 통해 수익을 창출하려는 프로 웹소설 작가.

## 3. 핵심 기능 요구사항 (Key Features)

### 3.1. World Bible (기억 및 설정 관리) - *The Memory*

- **기능 정의:** 인물, 장소, 사건, 아이템 등 고유 명사와 설정을 저장하고 관리하는 DB.
- **상세 요구사항:**
    - **인물 카드:** 이름, 외모, 성격, 말투(샘플 대사), 주요 관계를 입력.
    - **동적 업데이트:** 집필 중 새로운 설정(예: 주인공이 새 스킬 획득)이 발생하면 AI가 이를 감지하여 설정집 업데이트 제안.
    - **벡터 검색(RAG):** 집필 시 현재 씬에 등장하는 인물/장소와 관련된 설정을 사이드바에 자동 표시.

### 3.2. Plot Architect (구조 설계) - *The Director*

- **기능 정의:** 전체 시놉시스 -> 에피소드 -> 챕터 -> 씬(Scene)으로 이어지는 계층적 구조 설계 도구.
- **상세 요구사항:**
    - **트리 뷰(Tree View):** 드래그 앤 드롭으로 에피소드 순서 변경 가능.
    - **비트 시트 생성:** "회귀물 1화 국룰 전개로 짜줘" 요청 시, [주인공 죽음 -> 회귀 -> 각성]의 비트 시트 자동 생성.

### 3.3. Smart Editor (집필 및 생성) - *The Brain*

- **기능 정의:** 실제 글을 쓰고 AI와 협업하는 에디터.
- **상세 요구사항:**
    - **이어 쓰기 (Continue):** 커서 위치에서 다음 문장 3~5개 추천 생성.
    - **인필링 (In-filling):** 문장과 문장 사이를 자연스럽게 연결하는 묘사 추가.
    - **지문/대사 변환:** "A가 B에게 화내며 거절하는 내용"이라고 쓰면 실제 소설 대사로 변환.
    - **단어/문장 교정:** "판타지 풍으로", "무협 풍으로" 문체 변환 기능.

### 3.4. Analysis & Feedback (분석)

- **기능 정의:** 현재 글의 품질과 개연성 체크.
- **상세 요구사항:**
    - **설정 충돌 감지:** "이 캐릭터는 3화에서 죽었는데 왜 10화에 등장하나요?" 경고 알림.
    - **페이스 조절:** 대화 빈도와 서술 빈도를 분석하여 늘어지는 구간 지적.

## 4. 사용자 시나리오 (User Flow)

1. **Onboarding:** 장르(무협, 로판 등)와 문체 선택.
2. **Bible Setup:** 주인공 이름, 핵심 목표, 조력자 설정 입력.
3. **Outline:** AI와 채팅하며 1~5화 분량의 개요(Outline) 확정.
4. **Drafting:** 1화 씬(Scene) 별로 문장 생성 -> 작가가 수정 -> 확정.
5. **Review:** 설정 충돌 체크 후 최종 저장.
