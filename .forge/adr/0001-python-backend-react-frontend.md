# Python 백엔드 + React 프론트엔드 이원 스택

AI/RAG 오케스트레이션을 LangChain/LlamaIndex 등 성숙한 Python 생태계 위에서 다루기 위해 백엔드를 FastAPI(Python)로, 집필 에디터·실시간 UI가 핵심인 프론트엔드를 React/TypeScript로 분리한다. AI 파이프라인이 제품의 심장이고 향후 무거워질 것이 확실하므로, 단일 풀스택(Next.js)으로 가다 나중에 쪼개는 비용보다 처음부터 분리하는 비용이 더 작다고 판단했다.

## Considered Options

- **Next.js 풀스택** — 1인 개발 속도와 배포는 가장 빠르나, AI/RAG가 무거워지면 JS 생태계의 RAG 도구가 Python만큼 성숙하지 않아 결국 백엔드를 분리해야 할 위험이 컸다.
- **Python 백엔드 + React 프론트 (채택)** — 초기 구축 비용과 스택 이원화(인력·CI·배포 두 벌) 비용을 지불하는 대신, AI 파이프라인을 가장 강한 생태계 위에 올린다.

## Consequences

- 스택이 둘로 나뉘어 개발·배포·채용 비용이 증가한다. 1인/소수 팀에서는 컨텍스트 전환 비용이 실재한다.
- 프론트-백엔드 간 API 계약(스트리밍 포함)을 명시적으로 설계·유지해야 한다.
