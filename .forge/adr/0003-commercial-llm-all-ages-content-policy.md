# 상용 LLM API 하이브리드 + 전체이용가 콘텐츠 수위

집필·생성 AI를 상용 LLM API(Claude/GPT 등) 하이브리드로 구성하고(작업별 모델 스위칭), 생성 콘텐츠 수위는 "전체이용가(약 15세)" 수준으로 제한한다. 고수위 성인 웹소설(19금)은 명시적 비목표다. 무협의 살육·복수극의 잔혹 묘사처럼 일부 장르 표현은 상용 API의 콘텐츠 모더레이션에 걸릴 수 있으나, 수위를 전체이용가로 묶으면 정책 거절 리스크가 최소화되고 운영이 합법·단순해진다.

## Considered Options

- **상용 API + 전체이용가 (채택)** — 한국어 품질이 가장 높고 인프라가 단순. 수위 상한이 명확해 정책 충돌이 적다.
- **상용 API + 성인물 우회 경로** — 19금 시장을 노리려면 상용 API가 거절하므로 정책이 관대한 오픈/자체호스팅 모델이 필요. 인프라·운영·법적 난이도가 급증해 비목표로 둔다.
- **자체 호스팅 오픈소스** — 비용·데이터주권엔 유리하나 GPU 비용과 한국어 품질 리스크가 커 MVP에 과하다.

## Consequences

- 토큰 비용이 상용 API 단가에 종속된다 — 모델 스위칭과 비용 한도(rate/budget)를 반드시 설계해야 한다.
- 성인물 시장은 포기한다. 향후 진입하려면 모델·정책·연령인증을 별도 ADR로 재설계해야 한다.
- 특정 장르의 수위 높은 표현이 모더레이션에 걸릴 때의 사용자 대면 처리(완곡한 거절 안내)를 설계해야 한다.
