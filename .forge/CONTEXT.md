# StoryWeaver

AI가 작가의 세계관·캐릭터·설정을 기억하고 집필을 보조하는 웹 기반 웹소설 창작 SaaS의 도메인 용어집.

## Language

**작품 (Work)**:
한 명의 작가가 집필하는 하나의 소설 프로젝트. 모든 설정·플롯·원고의 최상위 소유 단위.
_Avoid_: 프로젝트, 책, 소설(파일)

**World Bible**:
한 작품의 모든 설정(인물·장소·사건·아이템 등)을 저장·관리하는 설정 저장소. "기억하는 AI"의 근거 데이터.
_Avoid_: 설정집, 위키, 노트

**엔티티 카드 (Entity Card)**:
World Bible 안의 한 항목(인물·장소·사건·아이템 중 하나)을 표현하는 정형 데이터 단위. 정해진 필드(이름·속성 등)를 가진다.
_Avoid_: 설정 항목, 노드, 문서

**타임라인 상태 (Timeline State)**:
한 엔티티가 특정 시점(챕터/씬)에 갖는 상태. "3화에서 사망", "5화에서 각성"처럼 시간에 따라 변하는 사실을 기록해 상태 추적과 설정 충돌 감지의 근거가 된다.
_Avoid_: 히스토리, 로그, 변경 이력

**씬 (Scene)**:
집필과 AI 생성의 최소 단위. `작품 → 에피소드 → 챕터 → 씬` 계층의 말단. (시놉시스는 이 계층의 한 단계가 아니라 작품 직속 요약 텍스트다 — 데이터 모델은 `docs/data-model.md` 참조.)
_Avoid_: 장면, 문단, 블록

**씬-엔티티 링크 (Scene-Entity Link)**:
특정 씬에 어떤 엔티티가 등장하는지를 명시하는 연결. 메모리가 무엇을 불러올지 정하는 1차 근거(벡터 유사도는 보조).
_Avoid_: 태그, 참조

**메모리 (Memory)**:
현재 집필 중인 씬과 관련된 World Bible 설정을 찾아 AI에게 컨텍스트로 제공하는 작업·그 결과. 정형 카드 + 벡터 검색 + 씬-엔티티 링크의 하이브리드로 구성된다.
_Avoid_: RAG (사용자 대면 UI·문구에서는 '메모리'를 쓰고 RAG는 피함; 내부 설계 문서에서는 RAG를 그대로 사용 가능), 컨텍스트 주입

**모델 스위칭 (Model Switching)**:
집필 작업의 종류(이어쓰기·교정·구조 생성 등)에 따라 저비용/고품질 LLM을 골라 호출하는 전략.
_Avoid_: 모델 라우팅, 폴백
