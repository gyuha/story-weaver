## 차이·현장 결정 (회고 입력)
교차검증이 잡은 3건은 모두 **경미(치명적 아님)** — 범위/사실 오류가 아니라 용어·표현 정합성 문제:

1. **[표현] "Smart Editor 3종" 라벨 혼동.** PRD 본문의 "World Bible + 메모리 + Smart Editor 3종"은 *대분류 3종*을 뜻하나, "Smart Editor 3종"으로 붙어 읽히면 Smart Editor 내부 기능 수(실제 4~5개: 이어쓰기·인필링·지문대사변환·문체변환·교정)와 충돌하게 들림. 범위 모순은 아님. → 라벨을 "대분류 3종" 또는 "핵심 3영역"으로 정정 권장.
2. **[용어] 시놉시스의 계층 위치.** CONTEXT는 `작품→시놉시스→에피소드→챕터→씬` 선형 계층으로 서술하나, data-model ER은 시놉시스를 작품 직속 1:1 속성으로 모델링(시놉시스가 에피소드의 부모가 아님). 데이터 모델 쪽이 합리적 → CONTEXT 용어집 문구 정정 권장.
3. **[일관성] 'RAG' 용어 사용.** CONTEXT는 '메모리'의 _Avoid_에 RAG를 두나, 설계 문서들이 'RAG'를 직접 사용. CONTEXT의 회피 지침은 "사용자 대면"에 한정이고 설계 문서는 내부 문서이므로 실질 위반은 아님 → 표면상 불일치, CONTEXT에 "내부 설계 문서 예외" 한 줄 보강하면 해소.
