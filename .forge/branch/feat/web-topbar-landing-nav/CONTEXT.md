# StoryWeaver — 브랜치 글로서리 오버레이

이 파일은 `feat/web-topbar-landing-nav` 브랜치에서 확정·교정한 용어만 담는다. 최상위 `.forge/CONTEXT.md` 위에 오버레이되며, 충돌 시 이 파일이 우선한다(ADR-0011).

## Language

**부 (Part)**:
한 작품을 큰 서사 단위로 나눈 묶음(예: 제1부 청운문편, 제2부 혈산문편). `작품 → 부 → 화(=챕터) → 씬` 계층에서 작품 바로 아래 단계다. `docs/data-model.md`가 정본 계층에서 **에피소드(Episode)**로 부르던 그 자리이며, 사용자 대면 명칭을 "부"로 통일한다(코드/DB 식별자 `episode*`는 당분간 유지 — `write`/'집필' 선례와 동일). web 코드에서는 각 챕터에 박힌 자유 문자열 `partLabel`로 표현된다.
_Avoid_: 에피소드(사용자 대면), 편, 장, 시즌

**화 (= 챕터 / Chapter)**:
부 안의 한 회차. 사용자 대면 명칭은 "N화"(예: 6화), 코드/도메인 식별자는 `Chapter`이며 `index`가 화 번호다. 다수의 [[씬]]을 가진다.
_Avoid_: 회차(코드), 에피소드

**버전 기록 (Version History)**:
한 [[씬]]의 본문이 **작가의 집필 시간축**을 따라 저장돼 온 스냅샷들의 기록. 시간대별로 과거 버전을 읽고, 현재 버전으로 덮어쓰며(현재로 보내기), 변경분을 비교할 수 있다. 편집은 현재 버전만 가능하고 과거 버전은 읽기 전용이다. **타임라인 상태와 다른 축이다** — 타임라인 상태는 엔티티가 *작중 시점*(3화 사망 등)에 갖는 상태이고, 버전 기록은 *원고가 실제로 고쳐져 온 이력*이다.
_Avoid_: 히스토리, 타임라인(타임라인 상태와 혼동 금지), 리비전

**관리자 (Admin)**:
서비스 운영자 역할(role `ADMIN`). [[계정 승인]]과 운영 통계 조회 권한을 가지며, 작품을 소유·집필하는 **작가**와 구분되는 별개 역할이다. 한 계정이 둘을 겸할 수 있으나 개념은 분리한다.
_Avoid_: 슈퍼유저, 운영자(코드 식별자), 매니저

**계정 승인 (Account Approval)**:
회원 가입한 계정에 [[관리자]]가 서비스 사용 허가를 부여하는 절차. 초기 테스트 기간 동안 승인된 계정만 서비스를 쓸 수 있게 하는 게이팅 수단이다. 계정 상태는 `대기(pending) → 승인(approved)` 또는 `거부(rejected)`로 전이한다. (이메일 인증·정지와 다른 축 — 이메일 인증은 본인 확인, 승인은 운영자 허가다.)
_Avoid_: 가입 승인, 활성화, 인증(이메일 인증과 혼동 금지)

**회원 (Member)**:
가입한 한 계정을 [[관리자]] 관점에서 부르는 말 — 승인 대상 목록·통계의 단위. 집필 맥락의 "작가"와 같은 사람을 가리키되, 관리 화면에서는 작품 소유자가 아니라 *승인·집계 대상 계정*으로 본다.
_Avoid_: 유저, 가입자
